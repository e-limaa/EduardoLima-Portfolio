Seguimos o [Versionamento Semântico 2.0.0](https://semver.org/) (SemVer). Isso garante que quem consome o Design System possa atualizar com confiança.

## Formato de Versão

Dada uma versão `MAJOR.MINOR.PATCH` (ex: `1.2.3`), incrementamos:

1. **MAJOR** versões incompatíveis (breaking changes).
2. **MINOR** funcionalidades novas compatíveis com versões anteriores.
3. **PATCH** correções de bugs compatíveis com versões anteriores.

## Política de Depreciação

Quando um componente será removido, seguimos um processo estrito:

1.  **Marcar como Deprecated**: Adicionar tag `@deprecated` e warning no console.
2.  **Período de Graça**: Manter por pelo menos uma versão MAJOR.
3.  **Remoção**: Acontece na próxima versão MAJOR.

<Callout type="warning" title="Mudanças Quebrantes">
  Breaking changes são permitidas **apenas** em versões Major. Nunca em Minor ou Patch.
</Callout>
