## Overview

Toggle represents a pressable on or off state for a single control.

## Usage

Use toggle when the pressed state itself is the main interaction, such as formatting, view mode or filter chips. Prefer button when the control triggers an action rather than maintaining state.

## Style

Keep the content short and recognizable. Toggles work best with short labels or familiar icons, especially when several controls appear together. Use the default variant for quiet inline controls and `variant="outline"` when the toggle needs clearer boundaries inside toolbars or dense settings clusters.

## Code

```tsx
import { Toggle } from "@limia/design-system";
```

## Accessibility

- Provide `aria-label` for icon-only toggles.
- Ensure the pressed state remains visible beyond color alone.
- Do not use toggle for destructive or ambiguous actions.
