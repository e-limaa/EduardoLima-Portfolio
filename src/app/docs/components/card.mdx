# Card

Cards contain content and actions about a single subject. They are flexible containers that grouped related information.

## Anatomy

Cards are composed of primitive slots that you can compose:
- `CardHeader`: Title and description container.
- `CardContent`: The main body.
- `CardFooter`: Actions area.

## Examples

## Usage Guidelines

<DoDontContainer>
  <DoDont type="do" title="Group related content">
    Use cards to separate distinct entities (e.g., a Project, a User, a Payment Method).
  </DoDont>
  <DoDont type="dont" title="Overload with actions">
    Keep the footer simple. If a card has too many actions, consider a dedicated page.
  </DoDont>
</DoDontContainer>
