## Color Roles

We assign specific roles to colors to communicate meaning. Using the correct role ensures consistency across the platform.

### Primary & Action
Used for the most important actions on a page. These colors should draw the user's focus.

## Usage Guidelines

### Accessibility

We guarantee WCAG AA contrast ratios for all standard text pairs.

<Callout type="warning" title="Contrast Check">
  Never rely solely on color to convey meaning. Always use text labels or icons alongside status colors to support colorblind users.
</Callout>

### Do's and Don'ts

<DoDontContainer>
  <DoDont type="do" title="Use Semantic Names">
    Use `bg-destructive` for error states or dangerous actions. This automatically handles red scaling and dark mode.
  </DoDont>
  <DoDont type="dont" title="Hardcode Hex Values">
    Avoid using arbitrary hex values like `#ff0000`. These break theming and require manual maintenance.
  </DoDont>
</DoDontContainer>
