## Overview

Alert communicates persistent inline feedback that should remain visible until the user has acknowledged it.

## Usage

- Use default alerts for informational messages that should stay attached to a page or form.
- Use `variant="destructive"` for urgent or blocking errors.
- Prefer alerts over toast when the message must remain visible in context.

## Style

Pair the alert with a short title and a compact description. Keep the message actionable and close to the element or section it refers to.

## Code

```tsx
import { Alert, AlertDescription, AlertTitle } from "@limia/design-system";
```

## Accessibility

- Alerts should be concise enough to announce clearly to assistive technologies.
- Do not rely on color alone to communicate severity.
- Keep the alert near the form or section that requires attention.
