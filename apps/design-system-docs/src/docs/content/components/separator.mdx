## Overview

Separator divides related content regions without adding visual noise.

## Usage

Use separator when grouping alone is not enough to distinguish adjacent regions. Avoid using it as a substitute for spacing or heading hierarchy.

## Style

Separators should be subtle and secondary. If the division feels too strong, the surrounding layout probably needs rhythm rather than more lines.

## Code

```tsx
import { Separator } from "@limia/design-system";
```

## Accessibility

- Use the correct orientation for vertical and horizontal separators.
- Do not rely on separator as the only sign of hierarchy.
- Keep related section titles and labels available to assistive technology.
