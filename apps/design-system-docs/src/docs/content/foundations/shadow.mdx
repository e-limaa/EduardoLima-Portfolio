## Overview

Shadow communicates elevation, separation and sometimes interactivity. In Limia it should support surface hierarchy, not decorate every element on the screen.

## Principles

Use shadow when an element needs to feel lifted from the current surface. Pair it with border, background and layering decisions instead of relying on blur or opacity alone.

## Implementation

- `shadow-sm`: subtle interactive lift or light separation from the background.
- `shadow`: default elevation for cards, menus and low-level overlays.
- `shadow-lg`: stronger lift for dialogs and critical floating surfaces.

## Accessibility

Shadow should never be the only cue that an element is interactive or elevated. Pair it with clear borders, backgrounds and focus states so the hierarchy remains perceivable.
