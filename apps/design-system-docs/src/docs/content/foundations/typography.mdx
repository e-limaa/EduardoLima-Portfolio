## Overview

Typography gives the interface hierarchy, pace and reading comfort. Limia uses a single sans-serif system family and a tokenized scale so teams can create contrast without inventing local text systems.

## Principles

Typography should communicate structure before decoration. Display styles establish entry points, heading styles organize sections and body styles optimize readability in dense product contexts.

<Callout type="info" title="One family, multiple roles">
  Keep hierarchy in size, weight and spacing. Do not introduce extra font families to solve emphasis problems that the scale already covers.
</Callout>

## Accessibility

Readable typography depends on size, contrast, line height and predictable hierarchy. Headings should reflect document structure, not only appearance.

<Callout type="warning" title="Structure matters">
  Do not choose a heading level only because it looks right. Keep semantic heading order and adjust visual style with the documented token scale.
</Callout>
