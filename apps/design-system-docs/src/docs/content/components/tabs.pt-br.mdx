## Visão geral

Tabs alternam entre visões relacionadas sem forçar navegação para fora do contexto atual.

## Uso

Use tabs quando as visões pertencerem ao mesmo contexto e a pessoa se beneficiar de alternância rápida. Prefira páginas separadas quando cada seção precisar de URL própria, hierarquia própria ou leitura longa.

## Estilo

Os labels devem ser concisos, comparáveis e escaneáveis na horizontal. Tabs demais geralmente indicam que a arquitetura da informação precisa de outra camada.

## Código

```tsx
import { Tabs, TabsContent, TabsList, TabsTrigger } from "@limia/design-system";
```

## Acessibilidade

- Tabs suportam navegação por teclado com setas quando focadas.
- Mantenha o estado ativo visível além da cor.
- Evite labels longos ou quebrados em múltiplas linhas dentro da lista de tabs.
