## Overview

Table displays structured data in rows and columns with clear hierarchy.

## Usage

Use table when users need to scan, compare or inspect structured datasets. Avoid it on small screens when a card or list pattern communicates the same information more clearly.

## Style

Keep column labels short, align values consistently and avoid overcrowding rows with unrelated actions. Tables work best when hierarchy is obvious before interaction.

## Code

```tsx
import {
  Table,
  TableBody,
  TableCell,
  TableHead,
  TableHeader,
  TableRow,
} from "@limia/design-system";
```

## Accessibility

- Use header cells for column and row context where appropriate.
- Do not rely on alignment or color alone to communicate meaning.
- Provide responsive alternatives when dense tables break down on mobile.
