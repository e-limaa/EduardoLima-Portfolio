## Visão geral

Botões disparam ações, confirmam decisões e conduzem o usuário por um fluxo. Use-os para enviar formulários, abrir overlays, confirmar mudanças de estado ou destacar um próximo passo claro.

## Uso

Use uma ação primária por região sempre que possível. Quando várias ações competirem, reduza a ênfase com `secondary`, `outline` ou `ghost`. Reserve `destructive` para ações irreversíveis ou de maior risco.

## Estilo

O estilo do botão comunica ênfase, não só marca. O tamanho deve refletir a densidade do contexto e a quantidade de texto. Use `link` quando a ação deve soar como navegação inline em vez de CTA de superfície. Use tamanhos de ícone apenas quando a ação for amplamente reconhecível ou já estiver rotulada por perto.

Os ícones dentro de `Button` herdam o mesmo token semântico de foreground do rótulo, então variantes preenchidas mantêm o contraste do ícone alinhado ao foreground ativo do botão. No Limia, esse ajuste é pensado para o componente `Icon` baseado em fonte, não para SVGs.

## Código

Use `Button` diretamente de `@limia/design-system`. Evite wrappers locais, a menos que você esteja compondo um comportamento específico de produto.

```tsx
import { Button } from "@limia/design-system";

export function Example() {
  return (
    <div className="flex gap-4">
      <Button>Criar projeto</Button>
      <Button variant="outline">Cancelar</Button>
    </div>
  );
}
```

## Acessibilidade

- Botões são acessíveis por teclado por padrão e devem manter foco visível.
- Botões só com ícone precisam de `aria-label` ou de um rótulo visível por perto.
- Labels destrutivos devem ser explícitos, como "Excluir projeto", em vez de genéricos como "Confirmar".
