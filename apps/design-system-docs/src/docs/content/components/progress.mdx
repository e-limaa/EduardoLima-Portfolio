## Overview

Progress shows task completion or loading progress over time.

## Usage

Use progress when completion can be estimated or measured. Prefer skeleton when the system is still loading structure and cannot provide a meaningful completion value.

## Style

Pair progress with nearby text when the number or outcome matters. The bar alone should support the message, not replace it.

## Code

```tsx
import { Progress } from "@limia/design-system";
```

## Accessibility

- Expose progress context in surrounding text when the numeric state matters.
- Do not use progress as the only feedback for a blocked or failed state.
- Keep determinate and indeterminate loading patterns visually distinct.
