## Overview

Tabs switch between related views without forcing navigation away from the current page.

## Usage

Use tabs when views belong to the same context and users benefit from quick switching. Prefer separate pages when each section needs its own URL, hierarchy or long-form reading.

## Style

Labels should be concise, comparable and horizontally scannable. Too many tabs usually indicate that the information architecture needs another layer.

## Code

```tsx
import { Tabs, TabsContent, TabsList, TabsTrigger } from "@limia/design-system";
```

## Accessibility

- Tabs support keyboard navigation with arrow keys once focused.
- Keep active state visible beyond color alone.
- Do not force long or wrapping labels into the tab list.
