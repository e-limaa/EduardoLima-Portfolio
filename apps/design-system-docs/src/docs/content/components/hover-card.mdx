## Overview

Hover card reveals lightweight contextual preview on hover or focus.

## Usage

Use hover card for supplementary metadata such as profile previews, labels or small reference details. Do not use it for critical information required to complete a task.

## Style

Keep the content short and skimmable. Hover card should feel like a preview, not like a full dialog or form.

## Code

```tsx
import {
  HoverCard,
  HoverCardContent,
  HoverCardTrigger,
} from "@limia/design-system";
```

## Accessibility

- Content must also be reachable via focus, not hover only.
- Do not place required actions or mandatory instructions exclusively inside the card.
- Keep trigger text meaningful outside the preview itself.
