## Overview

Section Header is the standardized introduction block for major page regions. It keeps headings, supporting copy and optional index metadata visually aligned across editorial sections.

## Usage

Use Section Header at the top of long-form sections such as Projects, Services or Testimonials. It is a composition-level component, so prefer it where a page needs a repeated introduction pattern rather than a generic heading primitive.

<DoDontContainer>
  <DoDont type="do" title="Keep the copy focused">
    Use a clear title and one supporting paragraph. Let the section body handle the deeper explanation.
  </DoDont>
  <DoDont type="dont" title="Stuff it with controls">
    Avoid turning the header area into a toolbar. If the section needs multiple actions, place them in the section body or an adjacent control row.
  </DoDont>
</DoDontContainer>

## Style

`align` changes the text layout while `index` and `label` add editorial metadata. Use `children` only when you need a custom trailing action or badge instead of the default metadata slot.

<Callout type="info" title="Children override the metadata slot">
  When `children` are provided, they replace the default index and label treatment. Use that escape hatch intentionally so the header remains predictable across the product.
</Callout>

## Accessibility

- Use `headingId` when the section body or wrapper needs `aria-labelledby`.
- Keep the title descriptive enough to stand on its own in screen-reader heading navigation.
- If `children` add interactive controls, make sure they remain keyboard reachable and do not replace critical heading content.
