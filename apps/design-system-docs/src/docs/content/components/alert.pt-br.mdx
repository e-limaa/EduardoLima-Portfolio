## Visão geral

Alert comunica feedback inline persistente que deve permanecer visível até a pessoa reconhecê-lo.

## Uso

- Use o alert padrão para mensagens informativas que precisam ficar ligadas à página ou ao formulário.
- Use `variant="destructive"` para erros urgentes ou bloqueadores.
- Prefira alert em vez de toast quando a mensagem precisar continuar visível no contexto.

## Estilo

Combine o alert com um título curto e uma descrição compacta. Mantenha a mensagem acionável e próxima do elemento ou seção a que ela se refere.

## Código

```tsx
import { Alert, AlertDescription, AlertTitle } from "@limia/design-system";
```

## Acessibilidade

- Alerts precisam ser curtos o bastante para leitura clara por tecnologias assistivas.
- Não dependa apenas da cor para comunicar severidade.
- Mantenha o alert próximo do formulário ou seção que exige atenção.
