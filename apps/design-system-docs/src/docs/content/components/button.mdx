## Overview

Buttons trigger actions, confirm decisions and move users through flows. Use them to submit forms, open overlays, confirm state changes or offer a clear next step.

## Usage

Use one primary action per region whenever possible. When several actions compete, reduce emphasis with `secondary`, `outline` or `ghost`.

<ComponentPreview>
  <div className="flex flex-wrap gap-4">
    <Button>Create project</Button>
    <Button variant="secondary">Save draft</Button>
    <Button variant="outline">Cancel</Button>
    <Button variant="ghost">Export</Button>
  </div>
</ComponentPreview>

## Style

Button styling communicates emphasis, not only brand. Size should reflect the surrounding density and the amount of text. Use icon sizes only when the action is widely recognizable or already labeled nearby.

## Code

Use `Button` directly from `@limia/design-system`. Avoid local wrappers unless you are composing product-specific behavior.

```tsx
import { Button } from "@limia/design-system";

export function Example() {
  return (
    <div className="flex gap-4">
      <Button>Create project</Button>
      <Button variant="outline">Cancel</Button>
    </div>
  );
}
```

## Accessibility

- Buttons are keyboard accessible by default and should remain focus-visible.
- Icon-only buttons must have an `aria-label` or nearby visible label.
- Destructive labels should be explicit, such as "Delete project" instead of generic "Confirm".

<DoDontContainer>

  <DoDont type="dont" title="Hide the intent">
    Avoid vague labels like "Submit" or "Click here" when the outcome is not obvious from nearby context.
