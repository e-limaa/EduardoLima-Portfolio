## Overview

Forms are the primary way users interact with the system. A well-designed form guides the user, minimizes errors, and creates a sense of confidence.

## Structure

Group related fields into sections or Cards. Use a clear primary action at the bottom right or left aligned depending on context.

## Validation

### Inline Validation
Show errors immediately when functionality allows (e.g., password strength), or on blur for format errors (email).

<DoDontContainer>
  <DoDont type="do" title="Clear Error Messages">

  </DoDont>
   <DoDont type="dont" title="Validate too early">
    Don't show an error while the user is still typing the first character.
  </DoDont>
</DoDontContainer>
