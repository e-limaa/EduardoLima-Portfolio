## Overview

Alert dialog confirms destructive or high-impact actions before execution.

## Usage

Use alert dialog when the user is about to trigger a change that is destructive, costly or hard to recover from. Do not use it for routine confirmation that slows the flow without adding safety.

## Style

Keep titles direct and consequence-focused. Users should understand what will happen and whether the action can be reversed before they choose.

## Code

```tsx
import {
  AlertDialog,
  AlertDialogAction,
  AlertDialogCancel,
  AlertDialogContent,
  AlertDialogDescription,
  AlertDialogTitle,
  AlertDialogTrigger,
} from "@limia/design-system";
```

## Accessibility

- The dialog traps focus while open.
- Title and description should explain consequence and recovery clearly.
- Destructive actions should have explicit labels such as "Delete project" instead of generic "Confirm".
