The Antigravity Design System is a managed dependency that provides a unified language for our portfolio products.

## Requirements

- **Node**: >= 18.0.0
- **React**: >= 18.0.0
- **Tailwind CSS**: >= 3.0.0

## Installation

Install the package alongside its peer dependencies:

```bash
npm install @antigravity/ds class-variance-authority clsx tailwind-merge @radix-ui/react-slot
```

---

## Setup

### 1. Import Global Styles

In your root layout or entry file (e.g., `main.tsx`, `app/layout.tsx`):

```tsx
import "@antigravity/ds/styles/globals.css";
import "@antigravity/ds/styles/tokens.css";
```

### 2. Configure Tailwind

Update `tailwind.config.js` to inherit the system's design tokens.

```js
// tailwind.config.js
import { antigravityPreset } from "@antigravity/ds/tailwind";

export default {
  content: [
    "./src/**/*.{ts,tsx}",
    "./node_modules/@antigravity/ds/dist/**/*.{js,mjs}"
  ],
  presets: [antigravityPreset],
  darkMode: ["class"],
  // ... other config
};
```

Once installed, you can use components immediately.

```tsx
import { Button } from "@antigravity/ds";

export default function App() {
  return (
    <div className="flex gap-4">
      <Button variant="default">Hello World</Button>
      <Button variant="secondary">Secondary Action</Button>
    </div>
  );
}
```

## Common Mistakes

<DoDontContainer>
  <DoDont type="do" title="Use System Components">
     Always import primitives like `Button` or `Input` directly from `@antigravity/ds`. This ensures you get the latest accessibility fixes and design updates.
  </DoDont>
  <DoDont type="dont" title="Create Local Wrappers">
    Do not wrap DS components in local "convenience" wrappers (e.g., `MyButton`). This breaks the update path and doubles the maintenance burden.
  </DoDont>
</DoDontContainer>

<DoDontContainer>
  <DoDont type="do" title="Use Token Classes">
     Use standardized sizing tokens like `w-9`, `h-9` or `p-4`.
  </DoDont>
  <DoDont type="dont" title="Use Arbitrary Values">
    Avoid magic numbers like `w-[37px]`. If a token doesn't exist, question if the design deviation is necessary.
  </DoDont>
</DoDontContainer>
