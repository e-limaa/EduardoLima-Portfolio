## Visão geral

`Icon` renderiza Google Material Symbols Outlined com cantos arredondados por meio dos estilos compartilhados do Limia. Use-o quando precisar de um primitivo único de ícone que herde cor e escale com a tipografia.

## Uso

Passe o nome de ligature publicado pelo Google Fonts, como `search`, `add`, `close` ou `arrow_forward`. Mantenha os ícones decorativos por padrão e adicione nome acessível no controle pai quando o ícone for a única affordance visível.

## Estilo

Os ícones devem herdar a cor do texto ao redor e se alinhar à ênfase semântica do componente pai. Evite misturar famílias de ícones no mesmo fluxo. Em superfícies de produto, prefira usar `Icon` dentro dos controles do Limia em vez de soltar texto de ligature bruto.

## Código

Importe `Icon` de `@limia/design-system` e componha-o dentro de componentes como `Button`, triggers de `Select` ou empty states.

```tsx
import { Button, Icon } from "@limia/design-system";

export function Example() {
  return (
    <Button size="icon" variant="outline" aria-label="Buscar">
      <Icon name="search" aria-hidden />
    </Button>
  );
}
```

## Acessibilidade

- Ícones decorativos devem permanecer com `aria-hidden`.
- Controles somente com ícone continuam precisando de nome acessível no pai, como `aria-label`.
- Não confie apenas no significado do ícone quando a ação ou o status exigir texto explícito.
