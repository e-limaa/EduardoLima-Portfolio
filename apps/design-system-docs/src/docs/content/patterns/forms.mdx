import { Button, Input, Card, CardHeader, CardTitle, CardDescription, CardContent, CardFooter, Label } from '@limia/design-system';

## Overview

Forms are the highest-friction interaction pattern in most products. Good form composition reduces ambiguity, groups related fields and keeps validation tied to the input that needs attention.

## When to use

Use this pattern when users need to enter or update structured data over more than one field. Prefer direct manipulation, toggles or single-purpose controls when a full form would add unnecessary friction.

<Callout type="info" title="Progressive complexity">
  Start with the smallest useful set of fields. Add advanced fields only when the task or user role genuinely requires them.
</Callout>

## Composition

Group related fields into sections and anchor the primary action at the end of the form. Long forms should expose rhythm through spacing, headings and help text.

## Code

Compose forms from public Limia primitives such as `Input`, `Label`, `Button`, `Checkbox`, `RadioGroup` and `Select`. Keep validation and submission logic in the product layer.

```tsx
import { Button, Input, Label } from "@limia/design-system";

export function ProfileForm() {
  return (
    <form className="space-y-4">
      <div className="space-y-1.5">
        <Label htmlFor="email">Email address</Label>
        <Input id="email" type="email" placeholder="name@example.com" />
      </div>
      <Button type="submit">Save changes</Button>
    </form>
  );
}
```

## Accessibility

- Keep labels visible and programmatically associated with inputs.
- Show validation close to the field that needs correction.
- Do not trigger error messaging on the first keystroke unless the control is explicitly real-time, such as password strength.

<DoDontContainer>
  <DoDont type="do" title="Explain the problem">
    Write errors that help users recover, such as "Password must contain at least 8 characters."
  </DoDont>
  <DoDont type="dont" title="Validate too early">
    Avoid showing an error while users are still entering the first characters of a field.
  </DoDont>
</DoDontContainer>
