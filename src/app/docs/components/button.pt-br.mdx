## Uso

### Primário (Padrão)
A ação mais importante em uma página ou região.

<ComponentPreview>
  <Button variant="default">Criar Projeto</Button>
</ComponentPreview>

### Secundário & Outline
Para ações de importância secundária ou para reduzir o peso visual quando múltiplos botões estão presentes.

<ComponentPreview>
  <div className="flex gap-4">
    <Button variant="secondary">Salvar Rascunho</Button>
    <Button variant="outline">Cancelar</Button>
    <Button variant="ghost">Exportar</Button>
  </div>
</ComponentPreview>

### Destrutivo
Para ações que removem ou destroem dados.

<ComponentPreview>
  <Button variant="destructive">Excluir Conta</Button>
</ComponentPreview>

### Tamanhos de Ícone
Botões podem ser dimensionados especificamente para ícones.

<ComponentPreview>
  <div className="flex items-end gap-2">
    <Button size="icon-xs" variant="outline" title="Ícone XS">
      <span className="w-3 h-3 block bg-current rounded-full" />
    </Button>
    <Button size="icon-sm" variant="outline" title="Ícone SM (igual sm)">
      <span className="w-4 h-4 block bg-current rounded-full" />
    </Button>
    <Button size="icon-md" variant="outline" title="Ícone MD (igual Input)">
       <span className="w-4 h-4 block bg-current rounded-full" />
    </Button>
    <Button size="icon-lg" variant="outline" title="Ícone LG (igual lg)">
      <span className="w-5 h-5 block bg-current rounded-full" />
    </Button>
  </div>
</ComponentPreview>

### Estado de Carregamento
Use o estado de carregamento para indicar que uma ação assíncrona está em progresso. Isso previne envio duplo.

<ComponentPreview>
  <Button variant="default" disabled>
    <span className="mr-2 animate-spin">⟳</span>
    Salvando...
  </Button>
</ComponentPreview>

## Acessibilidade

- **Suporte a Teclado**: Botões são nativamente acessíveis. Usuários podem navegar até eles via `Tab` e ativá-los com `Enter` ou `Space`.
- **ARIA**: Garanta que botões que não parecem botões (ex: ícones) tenham `aria-label` se não tiverem texto.
