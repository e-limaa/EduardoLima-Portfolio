## Visão geral

Acessibilidade é um requisito do sistema, não uma camada opcional. Documentação, componentes e guidance de adoção do Limia devem preservar interação inclusiva como parte do contrato do produto.

## Princípios

O Limia usa o modelo POUR como baseline operacional:

- **Perceptível**: a informação precisa ser apresentada de formas que o usuário consiga perceber.
- **Operável**: a interação precisa funcionar com teclado e tecnologia assistiva.
- **Compreensível**: labels, fluxos e erros devem ser claros e recuperáveis.
- **Robusto**: a semântica deve sobreviver entre browsers, devices e ferramentas assistivas.

## Implementação

### Teclado e foco

Todos os elementos interativos devem permanecer operáveis apenas com teclado. Não suprima o focus ring do sistema sem substituí-lo por uma alternativa igualmente visível.

### Semântica e labels

Prefira elementos HTML nativos e adicione ARIA apenas quando o texto semântico estiver ausente ou insuficiente.

```tsx
<button aria-label="Fechar menu">
  <XIcon />
</button>
```

### Cor, movimento e feedback

- Atenda ao contraste WCAG AA para texto padrão e estados de componente.
- Nunca dependa apenas da cor para comunicar status.
- Respeite `prefers-reduced-motion` para animações decorativas.
