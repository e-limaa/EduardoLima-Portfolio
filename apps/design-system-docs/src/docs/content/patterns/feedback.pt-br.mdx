## Visão geral

Feedback é o pattern para exibir progresso, sucesso, alerta e erro ao longo dos fluxos do produto. Um bom feedback reduz ambiguidade e torna o estado do sistema visível sem interromper o usuário além do necessário.

## Quando usar

Use este pattern sempre que a interface mudar de estado por causa de uma ação do usuário, processamento em segundo plano ou condição do sistema. Combine a superfície com a urgência: inline para feedback local, toast para confirmação global transitória, alert para mensagem contextual importante e dialog para decisões bloqueantes.

<Callout type="info" title="Escale com intenção">
  Comece pela superfície menos intrusiva que ainda comunica o problema ou resultado com clareza.
</Callout>

## Composição

Feedback de carregamento deve ficar próximo da ação que o disparou. Mensagens de sucesso e erro devem explicar tanto o resultado quanto o próximo passo quando houver recuperação necessária.

## Código

Monte feedback a partir dos primitives do Limia e mantenha a lógica de negócio na camada do produto. Use alerts para mensagem contextual, affordances de loading perto do controle que iniciou a ação e toasts para confirmações globais não bloqueantes.

```tsx
import {
  Alert,
  AlertDescription,
  AlertTitle,
  Button,
} from "@limia/design-system";
import { LoaderCircle } from "lucide-react";

export function SavePanel({ saving }: { saving: boolean }) {
  return (
    <div className="space-y-4">
      <Button disabled={saving}>
        {saving ? <LoaderCircle className="mr-2 h-4 w-4 animate-spin" /> : null}
        {saving ? "Saving changes" : "Save changes"}
      </Button>

      <Alert>
        <AlertTitle>Need attention</AlertTitle>
        <AlertDescription>
          Review the highlighted fields before publishing.
        </AlertDescription>
      </Alert>
    </div>
  );
}
```

## Acessibilidade

- Anuncie mudanças de estado de acordo com a urgência. Problemas críticos não devem depender apenas de toast.
- Mantenha feedback de loading próximo da ação iniciadora para que usuários de teclado e leitor de tela entendam o que mudou.
- Escreva mensagens de erro acionáveis, que ajudem na recuperação em vez de textos genéricos de falha.
