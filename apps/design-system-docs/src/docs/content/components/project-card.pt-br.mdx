## Visão geral

Project Card é um card de alto impacto para superfícies de portfólio e cases. Ele combina narrativa visual com imagem, motion, suporte a teclado e um acento de categoria alinhado à identidade da marca.

## Uso

Use Project Card quando o próprio card for a entrada para uma história mais rica, como um case study, projeto em destaque ou arquivo de trabalhos. Prefira cards mais simples quando motion, imagem em tela cheia ou feedback baseado em hover adicionarem ruído em vez de clareza.

<DoDontContainer>
  <DoDont type="do" title="Use para destaques navegáveis">
    Faça o card inteiro ser acionável quando ele abrir o detalhe do projeto, um case externo ou uma coleção filtrada.
  </DoDont>
  <DoDont type="dont" title="Forçar em views densas">
    Evite Project Card em listas administrativas compactas ou dashboards onde um card mais discreto ou uma linha de tabela comunica melhor.
  </DoDont>
</DoDontContainer>

## Estilo

O componente funciona melhor com imagens fortes, categorias curtas e uma única métrica de apoio. Mantenha o gradiente ligado à identidade do projeto, mas evite sobrecarregar o card com múltiplos badges ou blocos extras de metadados.

<Callout type="info" title="Qualidade da imagem">
  Use imagens com bom contraste e ponto focal claro. O componente já adiciona um overlay escuro, então evite imagens que já estejam muito escuras ou excessivamente filtradas.
</Callout>

## Código

`ProjectCard` é exportado por `@limia/design-system`. Hoje a prop `color` espera classes utilitárias de gradiente e deve continuar restrita a combinações de marca previamente definidas.

```tsx
import { ProjectCard } from "@limia/design-system";

export function WorkShowcase() {
  return (
    <ProjectCard
      title="Neo Bank App"
      category="Fintech"
      image="https://images.unsplash.com/photo-1563986768609-322da13575f3?q=80&w=1000&auto=format&fit=crop"
      metric="2024"
      color="from-violet-500 to-purple-500"
      onClick={() => console.log("Open project")}
    />
  );
}
```

## Acessibilidade

- Quando `onClick` é informado, o card expõe ativação por teclado com `Enter` e `Space`.
- Mantenha `title` descritivo, porque ele contribui para o nome acessível quando o card é interativo.
- Não esconda informação crítica apenas em hover. Categoria, título e métrica de apoio devem continuar visíveis em repouso.
