## Overview

Sheet opens supporting flows from the edge of the viewport while preserving page context.

## Usage

Use sheet for secondary editing, supportive settings and contextual tasks that benefit from staying anchored to the current page. Prefer dialog when the task is more critical than the underlying page.

## Style

Keep sheet content focused and vertically structured. The edge entry should feel like an extension of the current workflow, not a hidden full page.

## Code

```tsx
import {
  Sheet,
  SheetContent,
  SheetHeader,
  SheetTitle,
  SheetTrigger,
} from "@limia/design-system";
```

## Accessibility

- Sheet manages focus while open.
- Titles and descriptions should explain what the panel is for before the user reaches the form fields.
- Avoid overloading sheet with long multi-step flows.
