## Visão geral

Skeleton antecipa a estrutura de carregamento antes da chegada do conteúdo real.

## Uso

Use skeleton quando o layout já for conhecido e a pessoa se beneficiar de enxergar a forma do conteúdo que está chegando. Prefira progress quando a conclusão puder ser medida.

## Estilo

Faça o placeholder se aproximar do layout final. Skeleton deve reduzir a percepção de espera, e não introduzir uma estrutura diferente que mude demais quando os dados chegarem.

## Código

```tsx
import { Skeleton } from "@limia/design-system";
```

## Acessibilidade

- Skeleton apoia o carregamento, mas não o explica; mantenha uma mensagem real de status por perto quando o loading for significativo.
- Evite estados muito longos só com skeleton e sem outro feedback.
- A troca para o conteúdo real deve gerar o mínimo possível de layout shift.
