## Visão geral

Progress mostra o avanço de uma tarefa ou carregamento ao longo do tempo.

## Uso

Use progress quando a conclusão puder ser estimada ou medida. Prefira skeleton quando o sistema ainda estiver carregando estrutura e não puder fornecer um valor significativo.

## Estilo

Combine progress com texto próximo quando o número ou o resultado importarem. A barra deve apoiar a mensagem, não substituí-la.

## Código

```tsx
import { Progress } from "@limia/design-system";
```

## Acessibilidade

- Exponha o contexto do progresso em texto próximo quando o estado numérico importar.
- Não use progress como único feedback para um estado bloqueado ou de falha.
- Mantenha padrões de carregamento determinado e indeterminado visualmente distintos.
