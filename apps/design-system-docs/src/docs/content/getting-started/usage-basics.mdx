## Overview

Use Limia as a system, not as a parts bin. Adoption is successful when apps consume public exports, semantic tokens and documented patterns without recreating local UI layers.

## Setup

Start from the package entrypoints and shared styles:

- `@limia/design-system` for primitives and patterns
- `@limia/tokens` for token CSS and token references
- the shared Tailwind preset when the host app uses Tailwind

## Usage

Adopt the system in this order:

1. Replace local primitives with public Limia exports.
2. Remove hardcoded colors and spacing in favor of semantic tokens.
3. Keep product-specific composition in the app layer, not inside the design-system package.
