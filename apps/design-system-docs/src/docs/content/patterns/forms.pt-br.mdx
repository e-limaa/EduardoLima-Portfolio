## Visão geral

Formulários são o padrão de interação com maior atrito na maioria dos produtos. Uma boa composição reduz ambiguidade, agrupa campos relacionados e mantém a validação conectada ao input que precisa de atenção.

## Quando usar

Use esse padrão quando a pessoa precisar inserir ou atualizar dados estruturados em mais de um campo. Prefira manipulação direta, toggles ou controles de propósito único quando um formulário completo adicionar atrito desnecessário.

<Callout type="info" title="Complexidade progressiva">
  Comece com o menor conjunto útil de campos. Adicione campos avançados apenas quando a tarefa ou o papel do usuário realmente exigir.
</Callout>

## Composição

Agrupe campos relacionados em seções e ancore a ação principal ao final do formulário. Formulários longos devem expor ritmo por meio de espaçamento, títulos e textos de apoio.

## Código

Componha formulários a partir dos primitives públicos do Limia, como `Input`, `Label`, `Button`, `Checkbox`, `RadioGroup` e `Select`. Mantenha validação e lógica de envio na camada de produto.

```tsx
import { Button, Input, Label } from "@limia/design-system";

export function ProfileForm() {
  return (
    <form className="space-y-4">
      <div className="space-y-1.5">
        <Label htmlFor="email">Email</Label>
        <Input id="email" type="email" placeholder="name@example.com" />
      </div>
      <Button type="submit">Salvar alterações</Button>
    </form>
  );
}
```

## Acessibilidade

- Mantenha labels visíveis e associados programaticamente aos inputs.
- Mostre validação perto do campo que precisa de correção.
- Não dispare mensagens de erro no primeiro caractere digitado, a menos que o controle seja explicitamente em tempo real, como força de senha.
