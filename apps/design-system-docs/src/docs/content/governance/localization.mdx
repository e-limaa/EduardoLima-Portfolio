## Overview

Localization keeps Limia documentation usable in English and Portuguese without creating two diverging systems. The goal is structural parity, not loose translation.

## Policy

English remains the documentation source of truth, and Portuguese must preserve the same scope, chapter set and examples.

- translate titles, descriptions, body copy and example labels
- keep component names, prop names, code syntax and file paths unchanged
- never add sections or examples in one language that do not exist in the other

<Callout type="info" title="Parity is a release rule">
  Localization is part of documentation quality. Missing or structurally different files should be treated as a real docs defect, not as optional cleanup.
</Callout>

## Workflow

1. Write or update the English page first.
2. Mirror the exact chapter structure in the Portuguese file.
3. Translate editorial content while preserving code, imports and API names.
4. Run the workspace checks to catch missing pairs or structural drift.

When a translation is temporarily blocked, use the fallback only as a short-lived exception and document the gap clearly.
