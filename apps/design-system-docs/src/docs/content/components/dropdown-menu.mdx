## Overview

Dropdown menu exposes compact contextual actions and lightweight selection in a floating surface.

## Usage

Use dropdown menu for local actions tied to the current object or region. Avoid using it as a substitute for primary navigation or for long, complex option sets.

## Style

Keep labels concise, group actions when needed and separate destructive actions from routine ones. The menu should feel scannable in one pass.

## Code

```tsx
import {
  DropdownMenu,
  DropdownMenuContent,
  DropdownMenuItem,
  DropdownMenuTrigger,
} from "@limia/design-system";
```

## Accessibility

- Menus are keyboard navigable by default.
- Trigger labels should explain what kind of menu is being opened.
- Do not hide critical navigation paths inside contextual menus.
