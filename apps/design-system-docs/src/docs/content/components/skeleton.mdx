## Overview

Skeleton previews loading structure before real content arrives.

## Usage

Use skeleton when layout is already known and users benefit from seeing the shape of the incoming content. Prefer progress when completion can be measured.

## Style

Match the placeholder closely to the final layout. Skeleton should reduce perceived waiting, not introduce a different structure that shifts heavily when data loads.

## Code

```tsx
import { Skeleton } from "@limia/design-system";
```

## Accessibility

- Skeleton is supportive, not informative; keep real status messaging nearby when loading is meaningful.
- Avoid very long skeleton-only states without additional feedback.
- Layout shifts should remain minimal when real content replaces the placeholder.
