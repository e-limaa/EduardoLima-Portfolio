## Code Standards

- **Typescript**: All components must be strictly typed. Avoid `any`.
- **Tailwind**: Use our design tokens. Do not use arbitrary values e.g., `w-[350px]`.
- **Exports**: Export your component from `index.ts` to make it available to consumers.

<DoDontContainer>
  <DoDont type="do" title="Small API Surface">
    Keep prop counts low. Use `slots` or `asChild` composition pattern (coming soon) instead of boolean props for everything.
  </DoDont>
  <DoDont type="dont" title="Business Logic">
    Never include business logic (e.g., fetching data, specific analytics) inside the Design System. Components should be dumb and presentational.
  </DoDont>
</DoDontContainer>
