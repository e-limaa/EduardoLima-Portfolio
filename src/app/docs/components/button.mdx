## Usage

### Primary (Default)
The single most important action on a page or region.

<ComponentPreview>
  <Button variant="default">Create Project</Button>
</ComponentPreview>

### Secondary & Outline
For actions of secondary importance or to reduce visual weight when multiple buttons are present.

<ComponentPreview>
  <div className="flex gap-4">
    <Button variant="secondary">Save Draft</Button>
    <Button variant="outline">Cancel</Button>
    <Button variant="ghost">Export</Button>
  </div>
</ComponentPreview>

### Destructive
For actions that remove or destroy data.

<ComponentPreview>
  <Button variant="destructive">Delete Account</Button>
</ComponentPreview>

### Loading State
Use the loading state to indicate an asynchronous action is in progress. This prevents double-submission.

<ComponentPreview>
  <Button variant="default" disabled>
    <span className="mr-2 animate-spin">‚ü≥</span>
    Saving...
  </Button>
</ComponentPreview>

## Accessibility

- **Keyboard Support**: Buttons are natively accessible. Users can navigate to them via `Tab` and activate them with `Enter` or `Space`.
- **ARIA**: Ensure buttons that don't look like buttons (e.g. icons) have `aria-label` if they lack text.
