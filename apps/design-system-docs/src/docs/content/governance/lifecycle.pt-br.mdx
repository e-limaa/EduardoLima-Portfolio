## Visão geral

Ciclo de vida torna a maturidade explícita. Quem consome o sistema precisa saber se um componente ainda é exploratório, se já está pronto para adoção ou se entrou em trilha de depreciação antes de construir sobre ele.

## Política

O Limia usa quatro estados de ciclo de vida em exports públicos e documentação:

## Fluxo

A promoção deve ser orientada por evidência, não por tempo. Mover de `beta` para `stable` exige prova de que o componente funciona em documentação, API do pacote e adoção em produto.

- As checagens de acessibilidade passam para o modelo de interação pretendido.
- A documentação está completa em inglês e português.
- O componente é usado com sucesso em pelo menos um fluxo de produção.
- Os exports do pacote e os arquivos de lifecycle estão alinhados.

<Callout type="info" title="Regra de governança">
  Docs, lifecycle do pacote e adoção nos apps devem evoluir juntos. Um badge de stable não é só julgamento visual; ele também é um contrato de release.
</Callout>
