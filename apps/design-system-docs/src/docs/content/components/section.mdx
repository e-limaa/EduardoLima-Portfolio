## Overview

Section is the layout wrapper for long-form portfolio surfaces. It standardizes vertical rhythm, background treatment and optional container behavior so sections across a page feel related instead of individually styled.

## Usage

Use Section to define major page regions such as hero follow-ups, project lists, testimonials or contact areas. Keep business logic and section-specific storytelling inside the children rather than adding more layout-specific props to the wrapper.

## Style

`variant`, `grid`, `noise` and `container` let you tune the atmosphere without breaking consistency. The default recommendation is `variant="subtle"` with both `grid` and `noise` enabled for branded editorial pages.

        </p>
      </div>
    </Section>
  );
}

## Accessibility

- Section renders a semantic `<section>` element and should usually contain a visible heading or `aria-labelledby`.
- Decorative layers such as grid and noise must remain non-essential to comprehension.
- If the section is purely visual and has no landmark value, prefer a neutral wrapper instead of adding unnecessary page landmarks.
